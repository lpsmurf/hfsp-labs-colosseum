# Internal Documentation Guide

This folder is for internal team use only. It's not visible to external users or contributors.

## What Goes Here

✅ **Keep here:**
- Team meeting notes
- Internal decisions & discussions
- Brainstorming documents
- Experimental ideas
- Internal checklists & processes
- Work-in-progress documentation
- Team communication logs

❌ **Don't keep here:**
- Public documentation (use `/docs/`)
- API docs (use `/docs/API.md`)
- User guides (use `/docs/guides/`)
- Architecture docs (use `/docs/architecture/`)
- Contributing guidelines (use `CONTRIBUTING.md` in root)

## Organization

```
.internal/
├── team-comms/        Team meetings, decisions, communications
├── work-in-progress/  Experimental docs, brainstorms, drafts
└── checklists/        Internal processes, runbooks, checklists
```

## Access

This folder is in `.gitignore` (optional - set if you want to keep it private) or tracked but understood as internal-only.
