# Internal Documentation

This folder contains internal coordination, session notes, and agent communication documents.

**Not for external distribution.** These are working documents used by the development team for:
- Agent task coordination
- Session-to-session progress tracking
- Smoke test results
- Merge/deployment status
- Internal documentation

## Folder Structure

- `sessions/` — Session coordination docs, test results, progress reports
- Other internal reference docs as needed

## Note for Future Development

**Principle**: Keep the root `README.md` and core documentation public. Archive all internal coordination, test results, and agent communication docs in `.internal/` to maintain a clean public-facing repo structure.
